## ADDED Requirements

### Requirement: Claude exec SHALL support monitor-correlated status completion
`ccsm claude exec` SHALL be able to correlate a launched Claude execution with a monitor session using a stable run identifier generated by the CLI.

#### Scenario: Monitored exec starts
- **WHEN** the user starts `ccsm claude exec` with status-driven monitoring enabled
- **THEN** the CLI SHALL generate a unique run identifier for that invocation
- **AND** the CLI SHALL pass the run identifier to the launched Claude process environment
- **AND** the monitor SHALL be able to associate the resulting session with that run identifier or deterministic metadata

#### Scenario: Multiple exec invocations overlap
- **WHEN** two monitored `ccsm claude exec` invocations run concurrently in the same workspace
- **THEN** each invocation SHALL wait on and fetch outputs from only its correlated monitor session
- **AND** the implementation SHALL NOT choose a session only because it is the newest session in the workspace

### Requirement: Claude exec SHALL use monitor terminal state as the completion signal
`ccsm claude exec` SHALL treat correlated monitor session terminal state as the primary completion signal when status-driven monitoring is active and available.

#### Scenario: Session completes successfully
- **WHEN** the correlated monitor session status becomes `completed`
- **THEN** the CLI SHALL stop waiting for execution progress
- **AND** the CLI SHALL fetch final structured outputs for that session from the monitor
- **AND** the CLI SHALL return a successful command result unless a separate process or retrieval error requires failure

#### Scenario: Session ends in an error state
- **WHEN** the correlated monitor session status becomes `error` or `abandoned`
- **THEN** the CLI SHALL stop waiting for execution progress
- **AND** the CLI SHALL fetch any available structured outputs and session details
- **AND** the CLI SHALL report the terminal state clearly to the caller

#### Scenario: Claude process exits before hooks close the session
- **WHEN** the spawned Claude process exits before the correlated monitor session reaches a terminal state
- **THEN** the CLI SHALL continue checking the monitor for a bounded grace period
- **AND** the CLI SHALL either use the final terminal state observed during that grace period or report a clear timeout/fallback outcome

### Requirement: Claude exec SHALL retrieve structured final outputs after terminal state
After a correlated monitor session reaches terminal state, `ccsm claude exec` SHALL retrieve monitor-derived outputs without reducing them to a single assistant text by default.

#### Scenario: Agent Teams outputs are available
- **WHEN** the completed session contains outputs from multiple agents
- **THEN** the CLI SHALL preserve agent identifiers, latest output metadata, output counts, and output content in the retrieved result structure
- **AND** Codex or the calling model SHALL be able to inspect those outputs independently for acceptance

#### Scenario: Only fallback hook output is available
- **WHEN** transcript extraction yields no assistant messages but hook fallback output exists
- **THEN** the CLI SHALL retrieve and expose the fallback monitor output
- **AND** the command SHALL make clear that the output source is monitor-derived fallback data when that metadata is available

### Requirement: Existing Claude exec behavior SHALL remain compatible
The status-driven result path SHALL preserve existing Claude exec behavior when monitoring is disabled, unavailable, or explicitly configured for compatibility.

#### Scenario: Monitor is unavailable in compatible mode
- **WHEN** `ccsm claude exec` cannot reach a healthy monitor and strict status monitoring is not requested
- **THEN** the CLI SHALL use the existing process-exit behavior
- **AND** the command SHALL NOT wait indefinitely for monitor state

#### Scenario: Strict status monitoring cannot correlate a session
- **WHEN** strict status monitoring is requested and no correlated session is discovered before the configured timeout
- **THEN** the CLI SHALL fail with a clear diagnostic that includes the run identifier and monitor endpoint
- **AND** the CLI SHALL NOT silently return outputs from an uncorrelated session

#### Scenario: Agent Teams injection is disabled
- **WHEN** the user passes the existing option that disables Agent Teams environment injection
- **THEN** `ccsm claude exec` SHALL preserve that behavior
- **AND** the status-driven implementation SHALL NOT re-enable Agent Teams implicitly
